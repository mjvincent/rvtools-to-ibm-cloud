# ADR-008: Migration Readiness Assessment

## Status
Accepted

## Context
The application already produces IBM Cloud VPC Terraform, image readiness guidance, per-disk mapping, and multi-NIC mapping from RVTools exports. Customers still need an operational source-side readiness view before they schedule image export, replication, import, or cutover.

RVTools includes additional tabs that can expose common migration risks:
* `vSnapshot` for active snapshots and snapshot size.
* `vTools` for VMware Tools, heartbeat, application, and operation readiness status.
* `vCD` for mounted ISO or CD/DVD media.
* `vUSB` for attached USB devices.
* `vHealth` for health warnings when findings can be associated with a workload.

## Decision
Add an advisory migration readiness layer that assigns each VM a `Ready`, `Review`, or `Blocked` status. The assessment does not change Terraform output. It only enriches the dashboard and migration handoff files.

The generated ZIP now includes:
* Expanded `migration-manifest.json` records with a `migration_readiness` object.
* Expanded `vm-mapping.csv` readiness columns.
* New `readiness-findings.csv` with one row per finding.
* Updated `migration-runbook.md` remediation steps.

## Rules
`Blocked` is used for findings that should be resolved before export, replication, or import planning:
* Large active snapshot footprint.
* Connected ISO/CD media.
* Attached USB device dependencies.
* Severe health findings when available.

`Review` is used for findings that require owner validation:
* Smaller active snapshots.
* VMware Tools not running, old, upgradeable, or missing.
* Guest heartbeat, application status, or operation readiness concerns.
* Powered-off VMs that may need migration scope validation.
* Non-severe RVTools health warnings.

`Ready` means no migration readiness findings were detected from the available tabs. It does not guarantee application-level readiness or successful migration.

## Consequences
Migration teams get a practical cleanup list before cutover planning. The handoff remains tool-neutral and can be used by IBM Cloud custom image import workflows, RackWare-style tooling, custom scripts, or migration factory processes.

Because RVTools is metadata-only, the assessment cannot prove in-guest configuration, application consistency, snapshot age intent, or storage/file health. Findings are advisory and must be confirmed with VMware, application, and migration owners.
