# ADR-010: IBM Catalog Pricing Modes

## Status
Accepted

## Context
The original pricing model used a hardcoded IBM Cloud VPC profile list and hourly estimates. Earlier API experiments attempted to search IBM Global Catalog pricing artifacts directly by VPC profile name, such as `bx2-2x8`. That approach is fragile because VPC instance profile names are not guaranteed to be Global Catalog pricing artifact names or object IDs.

The app must keep working for offline demos and local assessment work, while still allowing future IBM Cloud catalog integration.

## Decision
Introduce a catalog pricing abstraction with three modes:
* `static`: bundled fallback profile and storage pricing.
* `cached`: load `data/ibm_vpc_pricing_cache.json` when available, including trusted IBM Global Catalog billing-dimension metadata produced by the supported cache generator.
* `live`: discover profiles from the IBM Cloud VPC API when `IBMCLOUD_API_KEY` is available.

The app now records pricing source, confidence, last-updated timestamp, and profile hourly value with each recommendation.

Live mode deliberately separates profile discovery from exact pricing. If live profiles can be discovered but exact pricing cannot be safely resolved, the app uses mapped static pricing where available and marks the confidence as `profile-live-price-static`.

Cached mode can now carry exact or partial IBM Global Catalog mappings. Exact catalog pricing is used only when a billing dimension maps to one positive, currently effective, linear metric. Partial mappings are labeled `cached_catalog`, and ambiguous or unmapped dimensions are preserved for diagnostics.

## Consequences
Positive:
* The app remains functional without credentials or network access.
* Pricing metadata is transparent in the UI and handoff files.
* Profile discovery can become live without pretending profile-name searches are exact pricing lookups.
* Cached pricing can be refreshed through the supported IBM Global Catalog cache generator without changing the sizing engine.

Negative:
* Live profile discovery does not yet provide exact IBM billing catalog prices.
* Cached pricing requires a trusted cache generation process and review of unmapped catalog diagnostics.
* Users must review pricing confidence before treating estimates as financial commitments.
