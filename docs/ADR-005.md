# ADR-005: Image Readiness Assessment

## Status
**Accepted / Implemented** (May 2026)

## Context
The migration handoff package maps VMware workloads to IBM Cloud VPC target infrastructure, but customers still need to know whether each VM is a straightforward image migration candidate. IBM Cloud VPC custom image import requires the image to be staged in IBM Cloud Object Storage, converted to a supported format, and kept within platform image size limits.

Terraform should remain focused on target infrastructure generation. Image migration readiness is a planning concern that should inform migration teams, and package preflight should stop builds when an in-scope VM has a `Blocked` image readiness status.

## Decision
We added an advisory image readiness assessment for each VM. The assessment uses RVTools metadata already available in `vInfo` and `vDisk`, including guest OS, firmware, disk count, power state, and inferred boot disk size.

Each VM is classified as:

1. `Ready`: No metadata blockers were found.
2. `Review`: Migration teams should validate metadata or operational prerequisites.
3. `Blocked`: The boot disk exceeds the IBM Cloud custom image size limit.

The assessment is shown in the Streamlit dashboard and table, exported in the migration manifest, mapping CSV, preflight report, and generated runbook, and used as a package preflight blocker when the status is `Blocked`.

## Consequences
* **Positive**: Migration teams can identify image import issues before cutover planning.
* **Positive**: The assessment remains tool-neutral and works with custom image import, replication tooling, or migration factory workflows.
* **Positive**: Terraform output remains stable for valid packages while preflight prevents known blocked image candidates from being bundled accidentally.
* **Negative**: The app cannot verify in-guest software such as `cloud-init` or `cloudbase-init`; those remain validation tasks for migration teams.
