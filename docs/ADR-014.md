# ADR-014: Advisory vPartition Storage Planning

## Status
Accepted

## Context
The application already preserves RVTools `vDisk` rows for boot/data disk planning and Terraform data volume generation. RVTools can also include `vPartition`, which adds partition-level capacity, consumed space, free space, and free percentage. That detail is useful for migration review, but it is not a direct replacement for virtual disk capacity when generating IBM Cloud block volumes.

## Decision
Preserve optional `vPartition` rows as advisory storage planning metadata.

Partitions are correlated to source disks by `Disk Key` when available. Rows that cannot be safely matched remain visible as unmatched VM-level partition records. Generated Terraform volume capacity, storage tier selection, and volume attachment behavior continue to use `vDisk` capacity.

## Consequences
Migration teams can review partition free-space and unmatched partition rows in the Storage tab, `partition-mapping.csv`, `disk-mapping.csv`, and the manifest without changing infrastructure generation.

This avoids unsafe automatic shrinking or reshaping of target volumes based on partition data that may not represent the full virtual disk layout.
