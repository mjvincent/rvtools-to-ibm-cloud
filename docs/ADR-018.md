# ADR-018: Pytest Fixtures and Text Snapshots

## Status
**Accepted / Implemented** (May 2026)

## Context
The repository previously kept production tests as top-level Python scripts with manual `if __name__ == "__main__"` runners. That worked for lightweight validation, but shared VM records, workbook builders, and generated output expectations were duplicated across files.

The project now has stable generated contracts for Terraform, CSV handoff files, image import variables, and migration manifests. Those contracts benefit from explicit review when they change.

## Decision
Move supported tests into `tests/` and run them with pytest. Shared records, workbook builders, and parsing helpers live in `tests/conftest.py`.

Use plain text snapshots under `tests/snapshots/` for selected generated outputs. Avoid a snapshot plugin for now so the workflow remains transparent and dependency-light.

Configure pytest to collect from `tests/` only. Research scripts under `experiments/` remain outside the supported test suite.

## Consequences
* **Positive**: The full suite can run with one `python -m pytest` command.
* **Positive**: Fixtures reduce duplicated setup and make new tests easier to write.
* **Positive**: Text snapshots make generated contract changes visible in code review.
* **Negative**: Snapshot files must be intentionally maintained when output contracts change.
