# ADR-013: Assessment Quality Report

## Status
Accepted

## Context
The application uses multiple RVTools worksheets for sizing, readiness, disk mapping, network mapping, and migration handoff output. Missing optional readiness worksheets do not create blockers, and some required planning areas can fall back to partial metadata from `vInfo`. Users need a clear way to see workbook coverage and confidence before using the generated outputs for migration planning.

## Decision
Add an advisory Assessment Quality report that records required and optional worksheet coverage, row counts, confidence, and planning impact.

The report is generated during RVTools parsing after worksheet cleanup and is passed through the Streamlit workbench, Terraform ZIP, and migration manifest. It includes:

- Required tab coverage for `vInfo`, `vDisk`, `vCPU`, `vMemory`, `vHost`, `vCluster`, and `vNetwork`.
- Optional readiness tab coverage for `vSnapshot`, `vTools`, `vCD`, `vUSB`, and `vHealth`.
- Inventory, storage, network, memory, migration readiness, and overall confidence.
- JSON and CSV handoff files for customer review.

## Consequences
The report improves transparency without changing the existing migration engine behavior. Confidence is advisory only: it does not alter VM readiness status, sizing recommendations, Terraform rendering, VM exclusion, or package build eligibility.

Missing optional readiness tabs lower migration readiness confidence but do not create `Blocked` findings. Missing `vDisk` or `vNetwork` can be marked as medium confidence only when fallback metadata exists in `vInfo`.
