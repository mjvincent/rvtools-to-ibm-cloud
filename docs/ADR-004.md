# ADR-004: Migration Handoff Package

## Status
**Accepted / Implemented** (May 2026)

## Context
The application already generates IBM Cloud VPC Terraform from RVTools exports, but Terraform alone does not move VMware images, perform replication, or orchestrate application cutover. Customers still need a structured handoff between infrastructure generation and the migration tooling or migration factory process that brings workloads into IBM Cloud Virtual Servers for VPC.

A RackWare-specific export would be useful for some customers, but it would couple the core application to one migration tool too early.

## Decision
We added a tool-neutral migration handoff package to the generated ZIP bundle. The package includes:

1. `migration-manifest.json`: Structured source-to-target VM mapping.
2. `vm-mapping.csv`, `nic-mapping.csv`, `disk-mapping.csv`, `partition-mapping.csv`, `memory-readiness.csv`, and `readiness-findings.csv`: Spreadsheet-friendly migration review data.
3. `assessment-quality.json` / `assessment-quality.csv`: RVTools worksheet coverage and confidence.
4. `preflight-report.json` / `preflight-report.csv`: Package safety blockers and warnings.
5. `pricing-diagnostics.json` / `pricing-diagnostics.csv`: Pricing source, billing-dimension, fallback, and unmapped metric diagnostics.
6. `image-import-variables.tfvars.example`: Placeholder map for IBM Cloud VPC custom image IDs.
7. `migration-runbook.md`: Generated runbook for image staging, Terraform apply, validation, and cutover.

The manifest includes source metadata, target Terraform mappings, profile and storage recommendations, image placeholders, migration wave placeholders, and assessment context such as contention and underutilization flags.

## Consequences
* **Positive**: Keeps Terraform focused on target infrastructure while adding a practical bridge to migration execution.
* **Positive**: Supports RackWare, custom scripts, IBM Cloud image import workflows, or migration factory processes without hardcoding one tool.
* **Positive**: Preserves right-sizing and business-case context for migration teams after the Terraform bundle is exported.
* **Negative**: Image conversion, replication, custom image import, and cutover automation remain external workflows until future adapters are added.
