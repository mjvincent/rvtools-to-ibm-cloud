# ADR-017: Standalone Trusted Pricing Cache Generator

## Status
**Accepted / Implemented** (May 2026)

## Context
The app can load `data/ibm_vpc_pricing_cache.json`, but previously did not provide a supported way to generate that cache. ADR-010 left this as follow-up work so cached pricing could be introduced without changing the sizing engine or Streamlit defaults.

Live pricing still separates profile discovery from exact IBM billing catalog pricing. IBM VPC profile names are not treated as exact Global Catalog pricing artifact identifiers.

## Decision
Add `scripts/generate_pricing_cache.py` as the supported cache-generation command. The script uses `IBMCLOUD_API_KEY` from the environment or an env file, discovers IBM VPC instance profiles for a selected region, applies the app's existing static mapped prices where available, and writes the cache through the existing cache writer.

Generated cache metadata uses `source=trusted-cache-generator` and `confidence=profile-live-price-static`. This identifies the cache as operator-generated and auditable while still being explicit that exact IBM billing catalog pricing is not claimed.

The Streamlit UI remains unchanged. Users generate the cache separately, then select `Cached IBM catalog` in the app.

## Consequences
* **Positive**: Cached mode now has a supported operational path.
* **Positive**: Offline and static fallback behavior remains unchanged.
* **Positive**: Pricing confidence remains transparent in the UI and handoff files.
* **Negative**: Exact IBM Global Catalog billing dimension mapping remains future work.
