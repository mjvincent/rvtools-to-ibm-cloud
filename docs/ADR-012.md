# ADR-012: Focused Module Split

## Status
Accepted

## Context
The application had grown around two large files: `app.py` for Streamlit UI, RVTools parsing, per-VM processing, dashboard rendering, and ZIP packaging; and `logic_engine.py` for sizing, readiness, Terraform rendering, and migration handoff output. The normalized VM data model made it possible to separate those responsibilities without changing public outputs.

The project still needs to preserve existing Streamlit columns, CSV headers, manifest fields, Terraform file layout, and tests that import from `logic_engine.py`.

## Decision
Split implementation into focused modules:

- `rvtools_parser.py` loads RVTools sheets, correlates tab data, and builds `MigrationVm` records.
- `assessments.py` owns image, migration, and memory readiness policy.
- `sizing.py` owns catalog profile selection, right-sizing, storage tier recommendation, and cost mapping.
- `terraform_renderer.py` owns generated Terraform root and module HCL.
- `handoff.py` owns manifest, CSV, tfvars example, and runbook generation.
- `ui.py` owns Streamlit table configuration and display helpers.

`app.py` remains the Streamlit orchestration entrypoint. `logic_engine.py` remains a compatibility facade that re-exports the public functions used by existing tests and callers.

## Consequences
The codebase now has clearer ownership boundaries for future parser, assessment, renderer, handoff, and UI work. The split intentionally preserves public output contracts and does not add runtime dependencies.

Compatibility imports can be retired later only after tests, documentation, and any external callers are migrated to the focused module paths.
